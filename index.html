Why it stays fast and cheap  ·  the context economics
Small context is built into the loop, not bolted on

Every cycle runs in a fresh, scoped session, never one conversation that balloons forever. The kickoff prompt, HANDOFF.md, and the pruned CLAUDE.md compress the handoff, so the next implementer starts lean instead of dragging the whole history forward.

Cheaper

Token cost scales with context size. Short sessions spend far fewer tokens for the same shipped work.

Faster and sharper

Lean context stays cache-friendly and avoids the quality decay that a bloated window causes.

You set the ceiling

A longer build buys more planning time. Keep plans tight and quickly buildable to keep every session small.
